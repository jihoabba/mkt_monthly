Step 01 · 문제 정의
01

불편함을
포착한다

자동화는 기술에서 시작하지 않습니다.
"이걸 매번 손으로 해야 하나?" 싶은 순간, 그 불편함이 곧 프로젝트의 씨앗이 됩니다.

기능 목록을 먼저 쓰지 않습니다. "이게 없어지면 얼마나 편해질까?"를 먼저 묻습니다.

"인스타 데이터 매주 복붙하는 데 30분씩 쓰고 있었는데,
이게 자동화 1순위라는 걸 알았습니다."
😤
반복 작업 발견
매주 같은 데이터를 같은 시트에 복붙하고 있다면, 그건 자동화 신호
🎯
해결 범위 정의
완벽한 시스템 말고, 지금 당장 고통을 줄이는 최소 기능부터
📝
한 문장으로 요약
"○○를 하면 ○○가 자동으로 된다" — 이게 명확해야 시작할 수 있다
Step 02 · AI와 설계
02

Claude에게
말을
건다

코드를 몰라도 됩니다. 마케터의 언어로 문제를 설명하면 Claude가 구조를 잡아줍니다.

어떤 기술 스택이 맞는지, 어떻게 데이터가 흘러야 하는지 — 함께 설계하면서 나도 이해합니다.

"인스타 API로 팔로워 수 뽑아서 구글 시트에 자동으로 넣고 싶어" — 이 한 마디로 시작했습니다.
🧠
Claude API · Anthropic
기획 의도 설명 → 구조 설계 → 코드 초안 생성. 마케터 언어를 개발 언어로 번역
🏗️
아키텍처 결정
Firebase냐 Sheets냐, Apps Script냐 Python이냐 — 상황에 맞는 스택을 같이 고릅니다
⚡
빠른 프로토타입
설계 → 코드 초안까지 보통 1시간 이내. 일단 돌아가는 것부터 만들고 다듬는다
Step 03 · 운영 및 수정
03

만들고
부딪히고
고친다

완성된 코드를 실무에 바로 투입합니다. 오류가 나면 에러 메시지를 그대로 Claude에 붙여넣고, 함께 고칩니다.

코드를 이해하는 만큼 더 잘 고칩니다. 그래서 복붙이 아니라 이해하면서 씁니다.

"돌아가는 게 완벽한 것보다 낫다. 부서지면 그때 고치면 된다."
🚀
실무 즉시 투입
완성도 70%여도 일단 씁니다. 현장 피드백이 가장 빠른 테스트
🔧
오류 = 다음 질문
에러 메시지 복붙 → Claude와 디버깅. 고칠 때마다 코드를 더 이해하게 됨
🔁
반복 개선
쓰면서 불편한 점이 보입니다. 그게 다음 버전의 요구사항이 됩니다
